## [已完成] 分红数据方案 B — 手动 + 自动合并（日期级去重）

- **[Change Log]** `backend/services/fip/portfolio_engine.py`：
  - `get_dividend_details()`：从 ISIN 级别互斥 → 日期级别去重
    - `manual_div_isins` 改为 `manual_dates_by_isin`（isin → set(date_str)）
    - 自动抓取现在对所有 ISIN 执行，但跳过已被手动录入覆盖的具体日期
    - 手动录入的 shares 改为使用发生日期的实际持仓（`get_shares_on_date`），更精确
  - `get_dividend_projections()`：从 Manual=Fallback → 双路并行 + 月度覆盖
    - yfinance 标准路径始终执行
    - 手动录入有数据时：外推未来事件，按月 key 覆盖 yfinance 预测（manual wins same month）
    - trailing_sum 为 0 时才从手动历史补充（不影响 yfinance 正常计算）
    - `source` 字段新增 `"hybrid"` 状态，区分纯自动/纯手动/混合三种情况
- **[Logic Reason]** 原设计互斥粒度太粗，录入1次手动就永久屏蔽该 ISIN 的自动数据。方案 B 允许更细粒度的控制，符合实际业务场景（如已公告分红用手动录入，历史分红继续用 yfinance）

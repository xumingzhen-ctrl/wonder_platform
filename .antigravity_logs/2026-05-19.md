# 2026-05-19 开发日志

## 功能迭代与 Bug 修复
1. **修复家庭财务体检工具汇率换算方向错误** (wonder-hub)
   - **问题原因**：之前的 API 获取固定以 CNY 为基准 (`latest/CNY`)，且在 UI 中的标签为 `[货币] 汇率`。当用户切换基准货币（例如 USD）时，用户习惯性地输入 `7.25`（认为 1 USD = 7.25 CNY），但系统由于固定汇率计算机制，将其视为 `1 CNY = 7.25 USD`，从而导致计算出的外币金额反而变大（方向相反）。
   - **解决方案**：
     - 修改 `Workspace.tsx` 中的 API 获取逻辑，从 `latest/CNY` 动态变为 `latest/${state.baseCurrency}`。
     - 将倒数逻辑从锚定 CNY 变为锚定当前的基准货币，确保获取到的外币汇率字典永远表示 `1 外币 = X 基准货币`。
     - 优化 UI，将 `[货币] 汇率` 修改为 `1 [货币] = ? [基准货币]`，杜绝理解歧义。
2. **修复代码合规性错误** (wonder-hub)
   - 移除 `next.config.ts` 中多余的 NextConfig 类型声明，防止 strict typechecking 报错。
   - 修复 `TopNav.tsx` 中 `onSuccess` 的类型定义，增加缺失的 `role?: string` 属性。
   - 在 `package.json` 的 `scripts` 中补充 `npm run typecheck` 脚本 (`tsc --noEmit`)。经测试目前项目类型检查完全通过。

## 2026-05-19 解释说明

用户反馈了截图，发现被动覆盖率是 10.3% 以及月度结余为 -89725。通过计算推导，这是因为用户本地浏览器的 LocalStorage 中还缓存着旧版本代码拉取到的错误汇率（即 USD = 0.14678）。
只需要用户点击『刷新』按钮，获取到修正后的 USD = 7.25 即可解决。

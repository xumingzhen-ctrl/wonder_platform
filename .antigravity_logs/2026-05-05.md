## [16:00] 修复投连险开关导致界面崩溃

**[Status]** 已完成

**[Change Log]**
- `apps/fis-hub/src/utils/ilpImpactUtils.js`: 
  - `applyILPToMCChart` 函数中 mc0_p50/p10/p90 改用严格 null+正数检测（防止 0 触发 ||1 但 null 不触发）
  - map 循环内新增 safe_p50/p10/p90 变量，对 null/非有限值安全回退
  - 月度循环中对 prev 数组元素同样加 prevSafe_ 防护
- `apps/fis-hub/src/components/ILPConfigPanel.jsx`: 
  - `fmtPct`/`fmt` 函数添加 `isFinite()` 防护，避免 NaN/Infinity 传入 toFixed() 引发崩溃

**[Logic Reason]**
MC chart 后端会将 NaN 替换为 null，当 p50/p10/p90 为 null 时，
null / mc0_p50 = NaN，进而感染 disc_p50 = disc_p50 × (1 - NaN) = NaN，
最终所有图表数据变为 NaN → Recharts 宽高计算得 -1 → React Minified Error #185 界面崩溃。

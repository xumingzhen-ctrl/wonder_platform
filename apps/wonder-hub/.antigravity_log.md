## [2026-04-04] 中文文章路由 Bug 修复 ✅

**[Status]** 已完成

**[Change Log]**
- `next.config.ts`：将 `output: 'export'` 改为仅在 `NODE_ENV === 'production'` 时生效，开发环境下不受静态导出的路由限制
- `src/lib/blog.ts`：重构 slug 提取逻辑，引入 `buildSlugMap()` 机制，优先读取 frontmatter 中的英文 `slug` 字段
- `src/content/blog/全球资产配置穿越周期的科学之道.md`：frontmatter 添加 `slug: "global-asset-allocation"`
- `src/content/blog/wonder-2026-asset-allocation.md`：frontmatter 添加 `slug: "wonder-2026-asset-allocation"`
- `scripts/convert-docx.mjs`：自动生成 slug 字段（格式：`wonder-report-YYYY-MM-DD`）

**[Logic Reason]**
Next.js `output: 'export'` 模式要求 `generateStaticParams` 必须预注册所有动态路由参数。中文 slug 在 URL 层面存在编码不一致问题（`%E5%85%A8` vs `全`），导致路由匹配失败。通过在 frontmatter 中强制使用英文 slug，将文件名与 URL 路径彻底解耦，是行业标准解法。

**[Pending/TODO]**

- 每次用 `npm run convert` 导入新文章后，需手动修改 `slug` 字段为有意义的英文标识符
